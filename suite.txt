4. CE QUI MANQUE (Points critiques)
A. Manques CRITIQUES pour le SEO et Google News
1. Pas de sitemap News (/news-sitemap.xml)
Google News nécessite un sitemap spécifique ne contenant que les articles des 48 dernières heures. Ton sitemap actuel liste tout sans filtre temporel ni namespace news:.

2. Pas de tag <article:published_time>article:published_time dans le HTML
Le <meta property="article:published_time"> est déjà dans l'OG, mais il manque article:modified_time explicitement dans les <meta>.

3. Pas de page /politique-de-confidentialite dédiée
Tu rediriges vers mentions-legales, mais Google préfère une page séparée.

4. Les images viennent d'Unsplash
Pour Google News et le droit d'auteur, utiliser des images stock sans licence claire est risqué. Il faut :

Soit des images libres de droits avec attribution
Soit tes propres visuels
Soit un service comme Pexels/Pixabay avec les bons crédits
5. Le contenu est du HTML brut dans content
Tu utilises dangerouslySetInnerHTML — c'est un risque de sécurité XSS si le contenu n'est pas sanitisé. Pour la génération automatique, il faut impérativement un sanitizer.

6. Pas de système de compteur de vues
La section "Les plus lus" récupère les articles les plus récents, pas les plus lus. Il manque un champ view_count ou une table article_views.

7. Pas de liens internes dans le contenu des articles
Les articles HTML n'ont aucun lien interne vers d'autres articles NoMask. Le maillage interne est crucial pour le SEO.

C. Pipeline de génération automatique (à implémenter)
[1] TRENDS FINDER
    → Google Trends API / RSS concurrents / X trending
    → Sélectionne 5-10 sujets/jour

[2] ARTICLE GENERATOR
    → Prompt IA structuré (titre SEO, contenu HTML, excerpt, tags)
    → Validation : longueur, structure H2, mot-clé, liens internes
    → Sanitization HTML (DOMPurify)

[3] IMAGE FINDER
    → Recherche images via Unsplash/Pexels API
    → Upload dans Supabase Storage
    → Génère l'URL et le caption

[4] REVIEW QUEUE
    → status = 'draft' par défaut
    → Dashboard admin pour valider/modifier avant 'published'
    → Ou publication automatique si score qualité > seuil

[5] PUBLISHER
    → Met status = 'published' + published_at = NOW()
    → Notifie Google via API Indexing (ping sitemap)
    → Post automatique sur réseaux sociaux
